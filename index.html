3. LES TROIS PILIERS DE L'INFRASTRUCTURE

3.1. PILIER 1 : AUTHENTIFICATION ET DNS

Le contrôleur de domaine permet à chaque collaborateur de s'authentifier une seule fois pour accéder à l'ensemble des ressources (ordinateur, fichiers, applications).

Le service DNS interne assure la résolution des noms :

  • Zone directe : ouzbek.global → transforme les noms en adresses IP
    • srv-pdc.ouzbek.global → 172.16.11.1
    • srv-web.ouzbek.global → 172.16.11.10
  • Zone inverse : permet de retrouver un nom à partir d'une adresse IP (utile pour les audits et la traçabilité)

3.2. PILIER 2 : SÉCURITÉ HTTPS ET CERTIFICATS

Pour garantir la confidentialité des échanges, une Autorité de Certification interne a été déployée :

  1. Création d'une autorité racine (CN=ouzbek-SRV-PDC-CA)
  2. Émission de certificats SSL pour les services web internes
  3. Déploiement automatique de l'autorité sur tous les postes via GPO

Résultat : Tous les accès aux interfaces d'administration se font désormais en HTTPS avec cadenas vert :

3.3. PILIER 3 : DONNÉES ET DROITS D'ACCÈS (AGDLP)

La gestion des droits d'accès suit le modèle AGDLP, standard des entreprises pour sa fiabilité et sa simplicité d'administration :

Utilisateur (compte individuel)
    ↓
Groupe Global (par métier : Direction, Commercial, RH...)
    ↓
Groupe Local (par ressource : partage Direction, partage RH...)
    ↓
Permissions sur les dossiers

Structure des partages implémentée :

\\\\srv-pdc\\Partages\\
├── Direction$     →  Accès : Direction uniquement
├── Commercial$    →  Accès : Équipe commerciale
├── Comptabilite$  →  Accès : Comptabilité
├── RH$            →  Accès : RH (données confidentielles)
├── Technique$     →  Accès : Équipe technique
└── Public         →  Accès : Tous les utilisateurs (lecture)
